Brief

In Mexico is a little tricky to recycle, actually is hard. Not sure why the government does no have a better way to help everyone to find places to recycle.

When I lived in Spain I remember that in every corner you could find recycle bins for every type of need. Although I know my country has a long way before can implement something similar I know also that there is a lot of people like me that want to help and be more responsible.

After some research I found some pages and documents about where you can find recycle centers, but those where all over the place, no order and more important no user friendly. So I decided to make this easier, found a document from a department of the government named “Semarnat” which had a lot of what I was looking for:


http://www.semarnat.gob.mx/archivosanteriores/transparencia/transparenciafocalizada/residuos/Documents/directorio_residuos.pdf 

This document has around 550 recycle centers divided by city or material, this was perfect. The problem? Was in PDF and there was no exact pattern.

I tried first to go manual and make a JSON file and copy paste every single element, I quickly gave up, was going to take me around 2 weeks to careful finish the process (and that was only one of the many steps) so I decided to go in another direction. I made a txt file, easier to manage, indent and create a pattern.

I also decided to create my own parser and convert the txt into a JSON and from that JSON file I could either move it to Mongo or use the file directly (more about why I used MongoDB later on)

You can see the gist of my parser here:


https://gist.github.com/Wilsonilo/d43b5f2f7a780db2fd81d3ec3a0360d7

After all of this, I used the same parser to ask Google for coordinates of each of the elements:



Funny thing, this part of the process had to do it in two different days, my limit was reached:



Once I had everything good In a JSON file I decided to use MongoDB because wanted to experiment with NoSQL, also pymongo makes everything easier.

Once I finished this I moved to the last 50% of the project:

1. Build the Flask routes / app to respond to calls to the db from the Javascript file in the index and community section

2. There are JS Functions that handle the votes from the users (increase the number of positives and negatives numbers in the MongoDB element)

3. Did a little of work to protect injection and limit the votes per user. For now the only protection is a system based on IP Address. The user can only vote once per day. Later on I may integrate registration

Once I finished the last parts of the Map and votes system I just deployed everything in Digital Ocean made a Gmail account to handle form from the Postulate sections (for consultation and new centers) I used a third library named yagmail for this.

The completion of the project took a week and a half.
